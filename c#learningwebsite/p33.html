Introduction to MVC

MVC stands for Model-View-Controller. It's a way to build web applications by separating the data (Model), the user interface (View), and the control logic (Controller).

Views in ASP.NET Core

Views are the part of MVC that handles the user interface. They use HTML, JavaScript, CSS, and Razor code. Razor is a special syntax for embedding server-side code into web pages.

Action Methods and ViewResults

Action methods in controllers return ViewResults to render views. A ViewResult uses a layout view, while a PartialViewResult does not.

Example Code


public class SampleController : Controller
{
    public ActionResult SampleAction()
    {
        return View();
    }
}

Razor Syntax

Razor syntax allows you to mix HTML with C# code. Here are some basic examples:

  • Variables: @foo
  • Loops: @for (int i = 0; i < 10; i++) {

    @i

    }
  • Code Blocks: @{ var foo = "bar"; }

Example Code


@model MyModel

@Model.Title

@for (int i = 0; i < 10; i++)
{
    
@i

}

Layouts

Layouts are like templates for your views. They contain common HTML that you want to share across multiple views.

Example Layout





    @ViewData["Title"]


    @RenderBody()

Partial Views

Partial views are reusable pieces of UI. They help keep your code DRY (Don't Repeat Yourself).

Example Partial View

Tag helpers are a way to add server-side code to your HTML tags. They make your HTML more readable and maintainable.
