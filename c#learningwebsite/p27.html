Chapter 27: WPF Graphics Rendering Services

In this chapter, you'll learn about the graphical rendering capabilities of WPF (Windows Presentation Foundation). WPF provides three main ways to render graphics: shapes, drawings, and visuals. Each method has its pros and cons.

Understanding WPFâ€™s Graphical Rendering Services

WPF uses retained-mode graphics, meaning it keeps track of the graphics you create and ensures they are redrawn correctly. This is different from older methods where you had to manually manage the graphics.

WPF Graphical Rendering Options

  • Shapes: Use the System.Windows.Shapes namespace to draw simple 2D shapes like rectangles, ellipses, and lines. These are easy to use but can consume a lot of memory.
  • Drawings and Geometries: Use the System.Windows.Media.Drawing classes for more lightweight 2D graphics. These are less interactive but use less memory.
  • Visuals: The most efficient way to render graphics, but it requires C# code and is not as user-friendly as shapes or drawings.

Rendering Graphical Data Using Shapes

Shapes are the easiest way to create interactive 2D graphics. You can use classes like Rectangle, Ellipse, and Line to draw shapes. These shapes can respond to user interactions like mouse clicks.

Rendering Graphical Data Using Drawings and Geometries

Drawings and geometries are more lightweight than shapes. They are used for non-interactive graphics and are defined using classes like GeometryDrawing and ImageDrawing.

Example: Drawing with Geometry

Exercise: Create a Drawing

Task: Draw a simple house using geometries.

Pseudocode:

1. Create a GeometryGroup.
2. Add a RectangleGeometry for the house base.
3. Add a PolygonGeometry for the roof.
4. Use a DrawingBrush to fill the geometries.

Rendering Graphical Data Using the Visual Layer

The visual layer is the most efficient way to render graphics. It is used for complex and high-performance graphics rendering. You need to use C# code to work with the visual layer.

Example: Drawing with Visuals

DrawingVisual visual = new DrawingVisual();
using (DrawingContext context = visual.RenderOpen())
{
    context.DrawRectangle(Brushes.Blue, null, new Rect(0, 0, 100, 50));
}

Exercise: Render a Visual

Task: Create a visual that draws a grid of squares.

Pseudocode:

1. Create a DrawingVisual.
2. Obtain a DrawingContext from the visual.
3. Use a loop to draw multiple rectangles in a grid pattern.
4. Close the DrawingContext.

Applying Graphical Transformations

Transformations allow you to rotate, scale, skew, or translate (move) graphics. You can apply transformations to any UI element.

Example: Rotating a Shape

Exercise: Apply Transformations

Task: Create a shape that rotates when a button is clicked.

Pseudocode:

1. Create a shape (e.g., Rectangle).
2. Add a RotateTransform to the shape.
3. Create a Button.
4. Add a Click event handler to the Button.
5. In the event handler, change the Angle property of the RotateTransform.

Summary

WPF provides powerful tools for rendering graphics. You can use shapes for simple, interactive graphics, drawings and geometries for lightweight graphics, and the visual layer for high-performance graphics. Transformations allow you to manipulate graphics in various ways.
