Simplified Guide to Razor Pages

  1. Introduction to Razor Pages

    Razor Pages are similar to MVC applications but are page-based. Each Razor Page has a .cshtml file (view) and a .cshtml.cs file (code-behind).

  2. Creating a Razor Page

    In Visual Studio, right-click the Pages directory, select Add ➤ Razor Page ➤ Razor Page - Empty. This creates two files: YourPage.cshtml and YourPage.cshtml.cs.

  3. PageModel Class

    The code-behind file (YourPage.cshtml.cs) inherits from PageModel. It contains methods to handle HTTP GET and POST requests:

    public class YourPageModel : PageModel {
    public async Task OnGetAsync() { }
    public async Task OnPostAsync() { }
}
  4. Data Binding

    Use properties in the PageModel to bind data to the view:

    public Car Entity { get; set; }
  5. Handling Requests

    GET request handler:

    public async Task OnGetAsync() {
    Entity = await _carDataService.FindAsync(6);
    return Page();
}

    POST request handler:

    public async Task OnPostAsync() {
    await _carDataService.UpdateAsync(Entity);
    return RedirectToPage("Index");
}
  6. View (Razor Syntax)

    Use Razor syntax to display data:

    @page
@model YourPageModel

    @Model.Entity.PetName

Exercise 1: Create a Simple Razor Page

  1. Create a new Razor Page

    Right-click Pages ➤ Add ➤ Razor Page ➤ Razor Page - Empty. Name it SimplePage.

  2. Update the PageModel 
    public class SimplePageModel : PageModel {
    public string Message { get; set; }
    public void OnGet() {
        Message = "Hello, World!";
    }
}
  3. Update the View 
    @page
@model SimplePageModel

    @Model.Message
