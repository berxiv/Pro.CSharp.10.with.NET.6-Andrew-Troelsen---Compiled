Chapter 30: Introducing ASP.NET Core

ASP.NET Core is a framework for building web applications using C# and .NET. It helps you create websites and services that can run on different operating systems like Windows, Linux, and macOS.

Key Concepts

  • MVC Pattern: Stands for Model-View-Controller. It's a way to organize your code:
    • Model: Represents the data.
    • View: Displays the data to the user.
    • Controller: Handles user input and updates the model and view.
  • RESTful Services: These are web services that follow certain rules to make them easy to use and understand.
  • Razor Pages: A simpler way to build web pages in ASP.NET Core.

Setting Up Your Project

  1. Create a Solution: A solution is like a container for your projects.
  2. Create Projects: You will create three projects:
    • AutoLot.Api: A RESTful service.
    • AutoLot.Mvc: A web application using MVC.
    • AutoLot.Web: A web application using Razor Pages.

Running and Debugging

You can run and debug your projects using Visual Studio or Visual Studio Code. This helps you find and fix errors in your code.

Features of ASP.NET Core

  • Controllers and Actions: Controllers handle user requests, and actions are methods in controllers that respond to those requests.
  • Routing: Determines how URLs map to controllers and actions.
  • Model Binding and Validation: Automatically converts user input into model objects and checks if the input is valid.
  • Filters: Run code before or after certain stages of request processing.

Exercises

Exercise 1: Create a Simple ASP.NET Core Project

Steps:

  1. Open Visual Studio.
  2. Create a new project using the "ASP.NET Core Web App" template.
  3. Name your project "MyFirstApp".

Pseudocode Hint:


1. Open Visual Studio.
2. Select "Create a new project".
3. Choose "ASP.NET Core Web App".
4. Name the project "MyFirstApp".
5. Click "Create".

Exercise 2: Add a Controller

Steps:

  1. Right-click on the "Controllers" folder.
  2. Select "Add" > "Controller".
  3. Choose "MVC Controller - Empty".
  4. Name it "HomeController".

Pseudocode Hint:


1. Right-click "Controllers" folder.
2. Select "Add" > "Controller".
3. Choose "MVC Controller - Empty".
4. Name it "HomeController".

Exercise 3: Create a View

Steps:

  1. Right-click on the "Views" folder.
  2. Select "Add" > "New Folder" and name it "Home".
  3. Right-click on the "Home" folder and select "Add" > "Razor Page".
  4. Name it "Index".

Pseudocode Hint:


1. Right-click "Views" folder.
2. Select "Add" > "New Folder" and name it "Home".
3. Right-click "Home" folder.
4. Select "Add" > "Razor Page".
5. Name it "Index".

Exercise 4: Add Routing

Steps:

  1. Open Startup.cs.
  2. In the Configure method, add a route for your controller.

Pseudocode Hint:


1. Open Startup.cs.
2. In Configure method, add:
   app.UseEndpoints(endpoints =>
   {
       endpoints.MapControllerRoute(
           name: "default",
           pattern: "{controller=Home}/{action=Index}/{id?}");
   });
