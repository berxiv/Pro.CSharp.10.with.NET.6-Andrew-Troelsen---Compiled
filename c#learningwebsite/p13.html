Chapter 13: LINQ to Objects

Introduction

LINQ (Language Integrated Query) is a feature in .NET that allows you to query data from different sources like arrays, collections, XML, and databases using a consistent syntax. This chapter focuses on LINQ to Objects, which means using LINQ to query in-memory data like arrays and collections.

Key Concepts for LINQ

Before diving into LINQ, let's review some important C# features that make LINQ possible:

  • Implicitly Typed Variables: Use var to let the compiler determine the type.
  • Object Initialization: Quickly create objects and set their properties.
  • Lambda Expressions: Short functions to pass as arguments.
  • Extension Methods: Add new methods to existing types.
  • Anonymous Types: Create objects without defining a class.

LINQ Basics

LINQ queries look similar to SQL but are used within C#. They are strongly typed, meaning the compiler checks them for errors.

Example: Querying an Array

Let's start with a simple example. Suppose we have an array of video game names and we want to find the ones that contain a space and sort them alphabetically.

string[] games = {"Morrowind", "Uncharted 2", "Fallout 3", "Daxter", "System Shock 2"};
var query = from game in games
            where game.Contains(" ")
            orderby game
            select game;

foreach (var game in query)
{
    Console.WriteLine(game);
}

Exercises

1. Filter and Sort Names

Task: Given an array of names, find those that start with 'A' and sort them.

Pseudocode Hint:

names = ["Alice", "Bob", "Andrew", "Charlie"]
query = from name in names
        where name starts with "A"
        orderby name
        select name

2. Find Even Numbers

Task: Given an array of numbers, find all even numbers.

Pseudocode Hint:

numbers = [1, 2, 3, 4, 5, 6]
query = from number in numbers
        where number is even
        select number

Advanced LINQ Features

Deferred Execution: LINQ queries are not executed until you iterate over the results.

Immediate Execution: Use methods like ToList() or ToArray() to execute the query immediately and store the results.

Example: Deferred Execution

int[] numbers = {10, 20, 30, 40, 1, 2, 3, 8};
var query = from number in numbers
            where number < 10
            select number;

foreach (var number in query)
{
    Console.WriteLine(number);
}

numbers[0] = 4;

foreach (var number in query)
{
    Console.WriteLine(number);
}

Exercises

3. Deferred Execution Example

Task: Modify an array after defining a LINQ query and see the changes in the results.

Pseudocode Hint:

numbers = [10, 20, 30, 40, 1, 2, 3, 8]
query = from number in numbers
        where number < 10
        select number
print query results
change numbers[0] to 4
print query results again

4. Immediate Execution Example

Task: Use ToList() to immediately execute a query and store the results.

Pseudocode Hint:

numbers = [10, 20, 30, 40, 1, 2, 3, 8]
query = from number in numbers
        where number < 10
        select number
resultList = query.ToList()
print resultList

Conclusion

LINQ to Objects allows you to query in-memory data using a consistent and readable syntax. By understanding the basics and practicing with exercises, you can effectively use LINQ in your applications.
